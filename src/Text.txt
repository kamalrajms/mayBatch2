## react intro

    What is React?

    1) React is a JavaScript library (not a framework) developed by Facebook (Meta) for building user interfaces (UIs), especially Single Page Applications (SPAs).

    2) It helps developers build reusable UI components that update efficiently when data changes.

    Instead of reloading a whole webpage when something changes, React updates only the parts of the UI that need to change.

    -------
    Why do we need React?

	It makes UI development faster, cleaner, and more efficient.

	Provides a better user experience with smooth updates without page reloads.

## usestate()

    usestate, is a react hook that allows you to add state (data that can change  over time)
    to a functional component.

    Normally ,  in js functions,variables disappear after the function run .But with
    usestate ,react remember values b/w re-render.

        syntax:

            const[state,setState]=usestate(initialValue)

        state---the current value of the state variables
        setState--- a function that updates the state
        initialValue--- the starting value of the state.


## useEffect()

    In react,useEffect is a hook that lets you perform side effect in a functional component.
    Side effect are task that happen outside the normal react rendering process ,like:

            Fetch data from an API
            Updating the browser DOM manually
            Starting/stoping timers.

    
    syntax:

        useEffect(()=>{
            //code



        },[dependency 1,dependency 2.....])

        []---- if that dependency is empty means  it run the 
        function only once at the begining.

## routing in react && nested routing.

    Routing means navigation b/w different pages in a web application without 
    refreshing the browser.

    In traditional websites ,clicking a link  reloads the whole pages.

    In react (single page application) ,routing lets us changes the view
    without reloading.

                better performance (no full page reloads)
                Smooth user exxperiance
                Control over navigation without server request.

    commend:npm i react-router-dom


    Structure of routing:

        <BrowserRouter>
            <Routes> 
                <Route path="/address path" elemelt={<Component/>}/>
                <Route path="/home" element={<Home/>}/>
                <Route path="/About" element={<About/>}/>
            </Routes>
        </BrowserRouter>

        BrowserRouter:

            It is the main wrapper component for routing in react app.
            it enables navigation using the browser URL (history API).
            Without BrowserRouter, routing will not working.

        Routes:

            it contains all the <Route/> component
            It tells react router which component to show for which address path.
            only one route will shown at a time.

        Route:

            Defines a specific URL path  and which component should load for the that
            path.

## usesearchParams()

    usesearchParams is a hook provided by react router  used to read,access and update query
    paramemeter (search paramemeter) in the URL.

    usesearchParams helps manage URL query paramemeter dynamically in react

    syntax:

        const[searchParams,setSearchParams]=usesearchParams();


        searchParams----- read values
        setSearchParams----- update values
